Communication Protocols

Ubiquisense proposes an abstract communication model decoupling transport from protocol, relying on full-duplex queuing infrastructure.

By selecting a communication protocol, you choose to only accept this particular protocol framing into a communication channel.

The communication channel will then be the tube transporting corresponding frames.

This model proposes several main communication protocols :

  • OSC
  • Firmata
  • DMX
  • Midi

OSC : Open Sound Control

This protocol is known as a Midi replacement. It is the Ubiquisense pivotal protocol. All information is expressed as '/' separated message with associated arguments.

DMX

Lighting protocol massively distributed worlwide. Ubiquisense mainly rely on OpenDMX APIs which are associated to the ENTTEC PRO USB devices.

Midi

This protocol is the worldwide standard for digital musical devices controllers. Ubiquisense offers a native routing for it.
